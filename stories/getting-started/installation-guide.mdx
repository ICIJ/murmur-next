# Installation Guide

The [International Consortium of Investigative Journalists](https://icij.org)
aims to build efficient and innovative tools for the biggest cross-border investigations in the world.
In order to build stable, consistent and user-friendly webapps, our team
started to define common components between projects, as well as best practices and
guidelines.

### Install the package

If you are using module bundlers such as Webpack or Vite, you might need to include the package into your project.
To get started, use NPM or Yarn to get latest version of **@icij/murmur-next**.

```bash
# with NPM:
npm i @icij/murmur-next

# or with Yarn:
yarn add @icij/murmur-next
```

### Importing components

`@icij/murmur-next` ships a tree-shakable ESM build. Import only the components
you use and your bundler drops the rest:

```js
import { ButtonIcon } from '@icij/murmur-next'

export default {
  name: 'component-name',
  components: {
    ButtonIcon
  }
}
```

Each component automatically imports its own styles, so the components you use
bring their CSS with them and unused components — and their CSS — never reach
your bundle. There is no separate Murmur stylesheet to import.

Murmur is a Bootstrap 5 design system: its components are styled on top of
Bootstrap 5, so your app still needs to provide Bootstrap 5's own CSS, as it
always has. Murmur's per-component styles only cover what Murmur adds on top.

If you would rather load every component's styles from a single file — for
example in a UMD/CDN page — the aggregate stylesheet is still published:

```js
import '@icij/murmur-next/dist/lib/murmur.css'
```

Importing the aggregate pulls in every component's styles at once, so it opts
out of per-component CSS tree-shaking.

### Using the global plugin

An alternative to importing individual components is to register Murmur as a
plugin in your app entry point. The all-in plugin registers every component at
once and loads their styles automatically:

```js
import { createApp } from 'vue'
import Murmur from '@icij/murmur-next'

const app = createApp({})

app.use(Murmur)
```

Now all components will be globally available in your app. Murmur also exposes a
configuration object to handle common configuration options. To customize the
default options please refer to the [dedicated page](#/utilities/config).

If you need to use a component with images, please make sure to
[configure assets](#/utilities/assets) properly so images are available in your
final app bundle.

> **Upgrading:** the ESM build now imports each component's CSS automatically,
> so a manual `import '@icij/murmur-next/dist/lib/murmur.css'` is no longer
> required for Murmur's component styles. The UMD/CDN build
> (`dist/lib/murmur.umd.cjs`) is unchanged.
